# Roadmap por níveis

Este diretório detalha a evolução do `apex-lakehouse` por níveis de maturidade. A ideia não é apenas listar tarefas, mas deixar claro o que muda em arquitetura, operação e valor de negócio a cada etapa.

## Sequência recomendada

1. [`level-1-foundation.md`](./level-1-foundation.md)
2. [`level-2-analytics.md`](./level-2-analytics.md)
3. [`level-3-operations.md`](./level-3-operations.md)
4. [`level-4-productization.md`](./level-4-productization.md)

## Regra de condução

Cada nível deve fechar:

- escopo funcional;
- qualidade mínima;
- observabilidade compatível com o próprio nível;
- documentação e critérios de aceite.

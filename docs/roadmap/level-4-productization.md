# Nível 4: Productization and Intelligence

## Objetivo

Evoluir de plataforma analítica robusta para produto de dados com experiências de consumo, alertas e diferenciação analítica.

## Escopo

- criar alertas de resgate anormal e desvio de comportamento;
- adicionar camada semântica explícita para métricas compartilhadas;
- introduzir cache seletivo para endpoints analíticos;
- expor endpoints mais ricos com filtros, paginação e limites de contrato;
- adicionar scorecards de qualidade e confiabilidade por dataset;
- documentar SLAs e SLOs por produto de dados.

## Entregáveis

- `gold.alertas_resgate_anormal`;
- catálogo de métricas com definições de negócio;
- APIs analíticas com paginação, filtros e cache;
- scorecards de qualidade e confiança;
- material de apresentação executiva do produto.

## Critérios de saída

- alertas com explicabilidade mínima;
- mesma definição de métrica em BI, API e documentação;
- contratos claros entre plataforma de dados e consumidores;
- backlog futuro de ML separado do núcleo operacional.

## Riscos se pular esta etapa

- consumo final fica limitado a exploração manual;
- valor percebido do projeto não acompanha a maturidade técnica construída;
- oportunidades de diferenciação ficam fora do produto.
